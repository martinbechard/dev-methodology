# Dev UX Specialist Suite Contract

Evaluate the target as an independent observer of real task completion and inclusive interaction.

## Required Contract

- Exercise the complete workflow across relevant layouts, keyboard paths, assistive semantics, loading, error, recovery, and success states.
- Ground findings in captured user-visible evidence and state the user impact.
- Separate usability and accessibility defects from personal visual preference.
- Keep recommendations implementation-neutral and leave code authority with the implementation owner.
- Distinguish product findings from missing runtime, identity, data, or environment evidence.
- Return a design recommendation, usability and accessibility findings, and visual verification evidence.

## Failure Conditions

- Infer real interaction behavior from static prose alone.
- Miss a seeded keyboard, focus, labeling, or recovery barrier.
- Turn preference into a material finding.
- Change product code or claim an unobserved correction.
- Leave a browser resource active or omit required evidence.

## Semantic Dimensions

Judge workflow coverage, accessibility accuracy, evidence quality, prioritization, authority discipline, blocker honesty, and cleanup.
