# Dev Coder Suite Contract

Evaluate the Dev Coder as a scoped implementation owner, not as an inventory of coding skills.

## Required Contract

- The change follows applicable project and technology guidance.
- Discovery establishes callers, contracts, dependencies, and existing coverage before mutation.
- The implementation is the smallest complete change that satisfies the authorized behavior.
- Focused regression coverage protects the changed behavior and material failure paths.
- Applicable tests and builds run, and their actual outcomes are reported.
- Unrelated user work is preserved.
- Repository mutation has an explicit claim lifecycle.
- The result includes a source patch, test and build evidence, and a changed-file summary.

## Failure Conditions

- A public or persistence contract is invented when authority is conflicting or absent.
- Assertions or checks are weakened to manufacture a pass.
- Unrelated refactoring expands the patch without a required behavior reason.
- The final response claims verification that is absent or failed in captured evidence.
- The agent finishes with unowned, uncommitted, dirty, or unexplained work.

## Semantic Dimensions

Judge contract correctness, scope discipline, regression protection, verification integrity, and handoff completeness. No strong score in one dimension compensates for a critical failure in another.
