# Methodology Maintainer Suite Contract

Evaluate the target as the end-to-end owner of coherent methodology source and derived state.

## Required Contract

- Inspect source, generated, documentation, adapter, script, and regression surfaces before editing.
- Change authoritative sources first and regenerate every affected derived artifact.
- Preserve stable names, source-versus-generated boundaries, unrelated work, and explicit deployment boundaries.
- Send the complete change to methodology-artifact-reviewer in a fresh context and own every accepted in-scope correction.
- Invoke dev-verifier only after review acceptance and repeat review after a verifier-driven correction.
- Stop after two failed attempts for the same finding or when required scope, authority, or dependency is unavailable.
- Commit or explicitly record no change, clean the worktree, release claims, and return status plus every evidence field.

## Failure Conditions

- Hand-edit generated output or change source to match stale derived state.
- Omit catalog, documentation, or regression surfaces affected by the source change.
- Use same-owner review, verify before acceptance, run dependencies concurrently, or exceed the correction cap.
- Expand unauthorized scope or deploy to user-home paths as verification.
- Report READY with drift, failed checks, dirty state, active claims, or missing independent evidence.

## Semantic Dimensions

Judge source ownership, alignment completeness, dependency sequencing, correction discipline, verification integrity, terminal status, and release hygiene.
