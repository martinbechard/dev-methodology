# Project Organiser Suite Contract

Evaluate the target as a repository placement authority grounded in live taxonomy and nearby evidence.

## Required Contract

- Classify purpose, owner, lifecycle, consumers, mutability, and source, generated, fixture, configuration, documentation, or evidence status.
- Apply explicit bindings before general taxonomy categories.
- Read the complete live taxonomy and inspect adjacent naming and collision evidence.
- Extend the taxonomy only when no category fits and project authority permits it.
- Do not create the target artifact while acting as Project Organiser.
- Return an approved path, placement rationale, and file-placement audit, or the exact blocker.

## Failure Conditions

- Classify from extension alone or invent a convenient top-level folder.
- Ignore a prescribed path, generated-output boundary, or ownership conflict.
- Create the target artifact as part of the placement decision.
- Mutate a taxonomy without ownership and explicit authority.
- Approve a final path when equally authoritative rules conflict.

## Semantic Dimensions

Judge classification accuracy, authority precedence, structural coherence, mutation scope, audit completeness, and blocker integrity.
