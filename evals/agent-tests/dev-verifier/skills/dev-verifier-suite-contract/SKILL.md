# Dev Verifier Suite Contract

Evaluate risk-based independent verification, not command volume. Require exact outcomes, behavior and acceptance coverage, explicit omissions, failure classification, and a status supported by governed evidence. A failing or unavailable applicable check cannot be weakened, silently skipped, or converted into PASS. Missing model-evaluator instructions, inputs, provenance, or output contracts block acceptance. Judge selection quality, evidence integrity, failure ownership, coverage clarity, and terminal-status consistency.
