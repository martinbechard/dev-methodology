Agent Roles and Skill Loadouts

This reference defines the portable agent roles and skill loadouts in the development methodology bundle. It is customer-ready: roles are generic, project-specific setup is documented in AGENTS-PLAN.md, operational routing comes from AGENTS.md files, and skills stay small enough to load only when the current job needs them.

Reference Model

1. Role

A role is an invokable job boundary with stable authority, inputs, outputs, and handoff expectations.

2. Router Reference

Root and nested AGENTS.md files describe project family, application tiers, paths, checks, data rules, and local terminology.

3. Skill Selection

The role loads only the methodology, stack, quality, wiki, or domain skills needed for the current request.

4. Runtime Adapter

Codex, Claude Code, Gemini CLI, Junie, or another host receives a runtime-specific role file or skill metadata.

5. Evidence

Review checklists, verification notes, handoffs, and run records make the work inspectable after completion.

AGENTS.md is the project-local selector. It should point the skills router to the right tiers, paths, validation commands, and data rules without copying the body of shared skills or redefining role authority.

Core Role Agents

Development Orchestrator

Coordinates multi-step work across agents while preserving bounded context, explicit handoffs, and verification evidence.

task decomposition context packs agent routing retry policy merge policy

Project Organiser

Classifies artifacts by purpose, chooses paths from the project taxonomy, applies path intent, and audits generated files after runs.

project taxonomy path intent nested AGENTS.md file placement audit

Project Agent Setup Agent

Creates AGENTS-PLAN.md so a project can validate role agents, skill loadouts, folder routing, nested plans, and AGENTS.md reference before normal work begins.

create-agents-plan development-methodology AGENTS-PLAN.md setup validation

Coding Agent

Implements scoped source changes using applicable AGENTS.md instructions, repository patterns, focused tests, and build commands.

careful-coding ast-grep stack skill packs fix-explanation

Code Review Agent

Reviews diffs for regressions, missing tests, weak verification, unsafe abstractions, rule violations, and documentation drift.

review-structured source evidence finding report fresh context

QA And Verification Agent

Runs acceptance checks, builds, unit tests, integration tests, E2E tests, lint gates, and artifact-specific verification.

validation test execution coverage matrix PASS WARN FAIL

Documentation Architect

Selects the right documentation route and loads one focused artifact creation skill for the requested methodology artifact.

development-methodology artifact creation source-backed docs page contract

Artifact Review Agent

Reviews finished methodology artifacts with an artifact-specific checklist and passes evidence to the shared page verifier.

artifact checklist documentation-page-verifier quoted evidence review report

Wiki Query Agent

Answers project questions from docs/wiki first, verifies against authoritative files, and records durable knowledge gaps.

project-wiki-query project-wiki-research code-project-wiki

Wiki Ingest Agent

Processes raw inputs into durable wiki leaves, digest entries, processed-source links, linted pages, and verifier evidence.

project-wiki topic writer topic verifier link leaves

Public Source Collector

Collects public raw source artifacts for approved topics, records exclusions, and leaves synthesis to wiki ingest.

raw-only boundary public evidence source windows

Backlog Steward

Creates, claims, resumes, blocks, completes, and archives typed backlog work with recoverable status evidence.

create-backlog manage-backlog agent-claim

Merge Coordinator

Integrates work from multiple agents or worktrees while preserving claims, ownership, verification, and conflict evidence.

agent-work-merge conflict routing branch evidence

Routing Rules

Signal Router action Example loadout
User asks for code change Use Coding Agent unless the task is primarily review, verification, documentation, or merge coordination. careful-coding, ast-grep, project-wiki-query, plus stack skills from AGENTS.md.
User asks for review Use Code Review Agent with fresh context and finding-first output. review-structured, relevant stack skill, source-reading tools, test evidence.
User asks for documentation artifact Use Documentation Architect and development-methodology to select exactly one creation route. development-methodology plus create-functional-spec, create-architecture, create-high-level-design, create-module-design, or create-project-wiki.
User asks to set up project agents or skills Use Project Agent Setup Agent to create AGENTS-PLAN.md and decide whether nested plans are needed. development-methodology, create-agents-plan, documentation-page-verifier, and documentation-bootstrap for first-time setup.
User asks for artifact review Use Artifact Review Agent with the matching artifact review skill and completed checklist evidence. review-architecture, review-functional-spec, review-high-level-design, review-module-design, or review-project-wiki.
Target files are in a specialized subtree Read the nearest nested AGENTS.md and add only the local skills, checks, and data rules named there. api-routes and clerk-auth under an API subtree; postgres-drizzle under a DB subtree; playwright under E2E tests.
Work spans multiple phases or agents Use Development Orchestrator to split the work and preserve handoff, claim, and verification records. structured-design, manage-backlog, agent-claim, agent-work-merge, review-structured.

Customer Deployment Contracts

AGENTS.md Is Local Reference

Root and nested AGENTS.md files select project family, tiers, paths, validation commands, and data boundaries. They do not duplicate shared skills.

Skills Stay Reusable

Skills contain portable procedures. They are loaded by role and task, not copied into project instructions or role prompts.

Roles Keep Authority Stable

New roles are added only when authority, output contract, lifecycle phase, context isolation, or runtime resource ownership changes.

Evidence Is Required

Reviews, verification, handoffs, and wiki updates should leave enough evidence for another agent or human to inspect the decision later.
