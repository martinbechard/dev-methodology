Interactive Agent And Skill Map

This diagram uses color to distinguish agent-to-skill assignments from agent-to-agent dependencies. Amber lines show skills an agent always uses, while violet lines show skills selected when an applicable rule matches. Blue arrows show direct dependencies between agents. Blue skill cards have a separate meaning: they identify technology skills selected during project setup. Technology Skills explains these skill-assignment conventions.

Skill Catalog

Dev Coder uses execute-workitem for delivery. The selected simple-workitem process ends with a verified local commit, while feature-branch-workitem creates a feature branch, pushes it, publishes a pull request through create-pull-request, and explicitly sets that pull request ready for review. Focused pull requests separate shared foundations from independently reviewable changes, stacked pull requests are created in dependency order, and draft status represents unfinished work rather than stack position.

Backlog persistence is a separate project selection. Dev Backlog Steward loads file-based-backlog or github-issues-backlog from the configured backend and asks for the missing selection instead of creating a second shadow queue.
