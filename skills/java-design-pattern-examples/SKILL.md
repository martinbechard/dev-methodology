# Java Design Pattern Examples

Combine with Java, Java Design, and the matching generic pattern skill.

## Example Routing

- Read [Java Creational Examples](references/creational-examples-java.md) for Abstract Factory, Builder, Factory Method, Prototype, and Singleton.
- Read [Java Structural Examples](references/structural-examples-java.md) for Adapter, Bridge, Composite, Decorator, Facade, Flyweight, and Proxy.
- Read [Java Behavioral Examples](references/behavioral-examples-java.md) for Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, and Visitor.
- Treat examples as minimal sketches, then preserve the project Java release, module boundaries, nullness, exceptions, concurrency, resources, and framework lifecycle.
- Prefer standard Java APIs and existing project conventions when they already embody the pattern intent.

## Verification

- Test observable contracts rather than example class shapes.
- Prove every lifecycle, failure, concurrency, and ownership decision introduced by the implementation.
