# Interpreter Pattern Review Checklist

- Question: Is a small domain language genuinely required?
- Question: Are grammar, precedence, associativity, context, results, and errors explicit?
- Question: Are parsing and evaluation responsibilities separated clearly?
- Question: Is syntax immutable and is evaluation state explicit?
- Question: Are recursion, input, time, memory, and side-effect limits enforced?
- Question: Do tests cover every production, invalid form, diagnostic, and limit?
- Question: Is the adoption threshold for parser tooling documented?
